Clawbernetes

Kubernetes-native orchestration for autonomous AI agent fleets

What Is It?

Clawbernetes is a Kubernetes operator that lets you deploy, manage, and observe fleets of AI agents using standard kubectl apply workflows. You declare agents as Custom Resources, and the operator handles pod creation, plugin configuration, LLM traffic routing, budget enforcement, and observability.

Architecture

You kubectl apply -f clawagent.yaml
↓
Clawbernetes Operator Go binary running in-cluster, watches 6 CRD types
↓
Agent Pod OpenClaw + observeclaw plugin
Gateway Pod PII proxy + classifier
Tempo + Grafana Traces, metrics, dashboards
Agent → observeclaw (budget/policy/anomaly) → Gateway (PII redaction) → api.anthropic.com
Agent → OTEL traces → Tempo → Grafana dashboard

The 6 Custom Resources

ClawAgent

A single autonomous AI agent. The operator creates a Pod running OpenClaw with identity files, skills, and the observeclaw plugin pre-configured.

Key fields: identity.soul model policy gateway observability skillSet connectors lifecycle

ClawGateway

Centralized LLM proxy. All agent traffic routes through it. Supports PII redaction via regex patterns and optional HuggingFace complexity classifier.

Key fields: topology port routing.evaluators[] anomaly webhooks[]

ClawPolicy

Budget limits and tool access controls. The observeclaw plugin enforces these inside each agent — no external enforcement needed.

Key fields: budget.daily budget.monthly budget.warnAt budget.downgradeModel toolPolicy.allow[] toolPolicy.deny[]

ClawSkillSet

Reusable skill packages. Each skill is a SKILL.md file mounted into the agent's workspace. Agents load skills at startup.

Key fields: skills[].name skills[].content

ClawConnector

External service connections. Defines how agents reach databases or HTTP APIs, with rate limiting and permission scoping.

Key fields: type driver host port credentialsSecret permissions[] rateLimit

ClawObservability

Deploys the tracing stack. Creates Tempo (OTLP collector) and Grafana (visualization) with a pre-provisioned fleet dashboard.

Key fields: tempo.enabled grafana.enabled otlpEndpoint otlpProtocol

How to Set Up the Infrastructure

Create the cluster and install CRDs 
make demo

Creates a local kind cluster named clawbernetes and installs all 6 CRD definitions.

Set your API key 
cp .env.example .env
# Edit .env and set ANTHROPIC_API_KEY=sk-ant-...

The key is injected into a Kubernetes Secret and mounted into every agent and gateway pod.

Build and deploy everything 
make demo-up

This single command builds the operator, builds the OpenClaw image (with observeclaw plugin), loads images into kind, creates secrets, starts the operator, and applies all sample CRs in dependency order:

# Applied in this order:
ClawObservability  # Tempo + Grafana (must exist before agents reference it)
ClawSkillSet       # Skills (agents reference by name)
ClawPolicy         # Budget + tool rules (agents reference by name)
ClawConnector      # External services (agents reference by name)
ClawAgent (x2)     # eng-agent + sales-agent
Deploy the proxy and access the UIs 
make demo-proxy
echo '127.0.0.1 eng-agent.local sales-agent.local grafana.local' | sudo tee -a /etc/hosts
kubectl port-forward svc/agent-proxy 8080:80 -n clawbernetes

Now open:

  • http://localhost:8080 — Fleet config dashboard
  • http://eng-agent.local:8080 — Engineering agent UI
  • http://sales-agent.local:8080 — Sales agent UI
  • http://grafana.local:8080/d/clawbernetes-fleet — Grafana traces dashboard

Sample: Full Agent Configuration

This is a complete agent definition. Every field is optional except metadata.name.

apiVersion: claw.clawbernetes.io/v1
kind: ClawAgent
metadata:
  name: eng-agent
  namespace: clawbernetes
spec:
  # --- Identity: who the agent is ---
  identity:
    soul: |
      You are a senior software engineer with deep expertise
      in distributed systems and Kubernetes.
    user: |
      Your operator prefers concise, technical responses.

  # --- Model: which LLM to use ---
  model:
    provider: anthropic
    name: claude-sonnet-4-6
    fallback:
      provider: anthropic
      name: claude-haiku-4-5

  # --- References to other CRDs ---
  skillSet: engineering-skills     # ClawSkillSet name
  policy: engineering-policy       # ClawPolicy name
  gateway: main-gateway            # ClawGateway name
  observability: fleet-observability # ClawObservability name
  connectors:
    - production-postgres           # ClawConnector name

  # --- Resources ---
  resources:
    requests: { memory: "1Gi", cpu: "1" }
    limits:   { memory: "4Gi", cpu: "3" }

  # --- Lifecycle ---
  lifecycle:
    restartPolicy: Always
    hibernateAfterIdleMinutes: 30
    maxRuntime: "24h"

Sample: Policy with Budget

apiVersion: claw.clawbernetes.io/v1
kind: ClawPolicy
metadata:
  name: engineering-policy
spec:
  toolPolicy:
    deny:
      - rm_rf
      - drop_database
      - kubectl_delete_namespace
  budget:
    daily: 10        # $10/day per agent
    monthly: 200    # $200/month per agent
    warnAt: "0.9"   # At 90%, downgrade model
    downgradeModel: claude-haiku-4-5

What Happens When You Apply

Operator resolves all references

Reads the ClawPolicy, ClawGateway, ClawObservability, and ClawSkillSet named in the agent spec.

Generates openclaw.json with observeclaw plugin config

Maps budget limits, tool deny lists, anomaly thresholds, and routing evaluators from the CRDs into the plugin's config schema. Registers a gateway-anthropic provider pointing at the ClawGateway URL.

Creates the agent Pod

Init container copies baked-in extensions (observeclaw plugin) and seeds workspace with SOUL.md, IDENTITY.md, skills, and config. Main container runs OpenClaw on port 18789.

Agent starts, plugins load

observeclaw: budget tracking, tool policy enforcement, anomaly detection, routing pipeline.
diagnostics-otel: exports traces/metrics/logs to Tempo.

LLM calls flow through the gateway

The routing pipeline runs evaluators in priority order (regex blocker → classifier → gateway proxy). The gateway handles PII redaction and forwards to the upstream LLM API.

Everything is observable

Every call is traced in Tempo with model, tokens, duration, and session info. The Grafana dashboard shows fleet-wide metrics in real time.
